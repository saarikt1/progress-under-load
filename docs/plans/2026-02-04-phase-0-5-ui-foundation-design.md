## Goal
Deliver a consistent UI foundation (Tailwind + shadcn/ui + Radix primitives) and render one placeholder chart on an exercise detail page, while preserving the current warm palette and rounded visual language.

## Key Decisions
- Install Tailwind CSS and use CSS variables as the source of truth for theme tokens.
- Use the shadcn/ui CLI to scaffold primitives and configuration.
- Keep the existing warm palette and rounded card language.
- Add a new exercise detail placeholder route at `/exercises/bench-press`.
- Add a single Recharts component with static dummy data on that page.

## Styling + Theme
- Global CSS (`src/app/globals.css`) remains the theme source using CSS variables.
- Tailwind config (or CSS-first setup as generated by the CLI) maps utilities to these variables.
- Use `components.json` to enable CSS variables and align aliases.
- Keep the current font stack and soft shadow language.

## UI Primitives
- Add shadcn/ui primitives: `Button`, `Input`, `Card`, `Dialog`, `Toast`.
- Components live under `src/components/ui` with `src/lib/utils.ts` for `cn` helper.
- Replace existing shell elements to use these primitives where relevant.

## Exercise Page + Chart
- Add `src/app/exercises/bench-press/page.tsx`.
- Render a `Card` with a simple Recharts line chart (static data, minimal tooltip).
- Use consistent spacing and typography with the rest of the shell.

## Tests (TDD)
- Add a test for the new exercise page that asserts the heading and chart container.
- Run tests to confirm red, implement, then rerun to green.

## Validation
- Dashboard cards use shared primitives and look consistent on mobile.
- `/exercises/bench-press` renders the placeholder chart.
- `npm test` passes.

## Non-Goals
- No real data plumbing or analytics logic.
- No authentication or database changes.
- No additional charts beyond the single placeholder.

## Implementation Notes
- Use shadcn CLI: `npx shadcn@latest init` and `npx shadcn@latest add ...`.
- Follow CLI prompts to set:
  - Global CSS path: `src/app/globals.css`.
  - Tailwind config path (as generated in this repo).
  - CSS variables enabled.
  - Aliases: `@/components`, `@/lib/utils`.
